Die Geschichte des Internets

1. Generation - Connection established!

Die erste Generation des Internets war der Durchbruch von DNS als "Telefonbuch" und der Aufbau von Webserver, welche Daten (mehrheitlich statische Seiten) um die ganze Welt verteilen konnten.

2. Generation - Mehr Dynamik

Als sich die Technologien weiterentwickelten, wurden immer dynamischere Websiten möglich (Vernetzung von Daten mittels APIs und unzählige weitere Webtechnologien). Dies führte zu dem, was die meisten Menschen heute als Internet bezeichnen. Animierte, schön gestylte Websiten mit dynamischem Inhalt. Trotz all den Errungenschaften des Web2, bleiben einige Schwachstellen bis Heute bestehen. Zum Beispiel ist es ziemlich einfach, eine Website zu zensieren und offline zu bringen. Dies vorallem, weil die IP Adressen der Webseiten über das zentralisierte DNS System aufgelöst werden. Diese können beschlagnahmt oder blockiert werden und eine Webseite wird unauffindbar. Wird noch der dazugehörige Facebook und der Twitter Account zensiert, verliert die betroffene Organisation die komplette Audienz und wird so gut wie unauffindbar.

3. Generation - Mesh Networks

Die dritte Generation des Internets wurde nicht zuletzt auch mit Bitcoin eingeleutet. Die unterliegende Technologie (Blockchain - oder einfach eine Datenbank mit DLT (Distributed Ledger Technology)) ermöglicht es, ein Netzwerk von Computer aufzubauen, die sich konstant synchronisieren und somit einen Konens über den Status der geteilten Daten erlangen. Dezentralisierte Netzwerke oder sogar Mesh Netzwerke wie IPFS (Inter Planetary File System) bieten die Möglichkeit, Applikationen oder auch Websiten denzentralisiert zu hosten. Diese Funktionalität kann die Zensur oder sogar auch staatliche Firewalls umgehen. Mit der Krypto-Community entstehen immer mehr Anwendungsfelder für solche Applikationen -> die Effizienz wird besser und die Performanz nimmt zu.

Inter Planetary File System (IPFS)

Inter Planetary... Was??

Das Inter Planetary File System ist eine Technologie aus der Welt des Web3. Es vereint mehrere Eigenschaften, die für eine Vielfalt von Applikationen nützlich sind. Der hier relevanteste Teil ist der Mechanismus, wie Dateien lokalisiert werden können.

Lokationsbasierte vs. Inhaltsbasierte Auflösung

Wie bereits erwähnt funktioniert das Web1 & Web2 bis heute über DNS (Domain Name System) und löst somit eine Domain wie zum Beispiel "zensurresistent.ch" auf die hinterlegte IP Adresse auf, wo der Webserver die Website ausliefert. Da diese IP Adresse immer nur ein einziges Ziel haben kann, gibt es genau einen einzigen Webserver, der den Request bearbeitet. Somit kann man sagen, dass die Suche Lokationsbasiert statt findet. Wir lösen den Name der Domain auf die IP Adresse auf, welche den genauen Ort der Webseite angibt.

Zurück zu IPFS... IPFS ist ein Netzwerk aus Computer, welche miteinander kommunizieren und gegenseitig Daten austauschen können. Dies nennt man in diesem Fall ein Peer-To-Peer Netzwerk. Es gibt also kein zentraler Bestandteil, der für die Funktionsweise zuständig ist. Jede Node ist unabhängig und verbindet sich mit anderen Rechnern.

Wenn nun eine Datei auf einenen dieser Rechner hochgeladen wird, wird diese Datei in kleine Blöcke zerlegt und bekommt einen Content Identifier (CID). Dieser CID Wird durch eine Kryptographische Funktion wie zb der SHA-265 Algorithmus generiert und ist für jede hochgeladene Datei einzigartig. Falls diese Datei nun über IPFS veröffentlicht wird, ist sie genau unter diesem CID auffindbar.

Wenn also nun ein anderer Computer (oder ein User eine IPFS Node nutzt und) diesen CID im IPFS Netzwerk sucht, werden verschiedene Nodes Teile der Datei liefern, welche dann lokal zu der kompletten Datei zusammengesetzt werden können. In diesem Fall spielt es also nun keine Rolle mehr, welcher Server die Datei liefert, oder wo genau sich dieser befindet, wichtig ist nur, dass er die richtige Datei hat. Wenn diese Datei eine Webseite ist, wird sie somit so gut wie zensurresistent. Weiter ist der Sicherheitsmechanismus inherent zum Protokoll. Niemand kann die gesuchte Datei auf dem Weg von einem zum anderen Computer insgeheim verändern, da sich ja automatisch der CID verändern würde und somit die Datei nicht mehr der gesuchten entspräche... Genial, nicht? ;)

Distributed Ledger Technology (aka Blockchain)

Transaktionen als Informationsträger nutzen.

Blockchains wie zb Bitcoin, basieren auf dem Konsens über den Status der Daten. In diesem Fall geht es um Transaktionen die in sogenannten Blocks aneinander gekettet werden. Ist ein Block einmal im Netzwerk akzeptiert (validiert & gemined), dann wird das gesamte Computernnetzwerk diesen BLock bei sich abspeichern. Da die Blöcke "aufeinander" aufbauen, können bereits validierte Blöcke nicht mehr verändert werden. Diese Eigenschaft ist eine der Sicherheiten von Bitcoin. Es können weder Coins einfach so erfunden, noch Transaktionen rückgängig gemacht werden. Somit können einzelne Transaktionen in diesen Blöcken auch nicht gelöscht werden. Die Informationen sind persistiert im gesamten Netzwerk & somit auch zensurresistent.

Memo Protokoll

Memo als zensurresistente Datenbank.

Die erwähnte zensursichere Ablage von Transaktionen auf einer Blockchain kann nun von Usern genutzt werden um Metainformationen abzuspeichern. Ähnlich wie bei einer Banktransaktion kann zb eine beliebige Nachricht an den Empfänger übermittelt werden.

Das Memo Protokoll ist so eine Form der Übermittlung von Metadaten. Das Memo Protokoll wurde auf der Bitcoin Cash (ein sogenannter hard fork von Bitcoin) Blockchain entwickelt und kann dank den niedrigen Transaktionskosten, zum speichern von Informationen verwendet werden.

Auf memo.cash sieht man eine konkrete Anwendung. Es ist eine Art Twitter, welche ihre Daten komplett aus BCH Transaktionen via Memo Protokoll speichert und lädt.

Permissionless Software Foundation (PSF)

Die Ziele einer DAO.

Die dezentralisierte autonome Organisation (DAO) Permissionless Software Foundation beschäftigt sich mit den Themen zensurresistenz und permissionsless software, wie der Name es schon sagt. Sie entwickelt Softwaremodule, welche die Nutzung dieser Technologien vereinfacht. Der hier verwendete Code ist Open Source und ist auf der Website der PSF einsehbar.

So funktionierts...

Auf dem normalen Webserver von zensurresistent.ch liegt eine Website, dessen einzige Aufgabe ist, auf die IPFS Webseite weiterzuleiten. Dies geschieht folgendermassen:
  1. Beim öffnen der Website wird nun im Hintergrund automatisch auf die BCH Blockchain zugegriffen. Die Information, welche von der Blockchain via Memo Protokoll geholt wird, ist der letzte veröffentlichte CID der Website.
  2. Sobald dieser CID gefunden wurde wird eine URL zusammengebaut, die auf den Cloudflare Gateway für das IPFS Netzwerk zeigt.
  3. Du wirst auf die IPFS Website weitergeleitet.
Die Website wird lokal entwickelt, also nichts Neues hier. Sobald ein Update veröffentlicht werden soll, wird ein kleines Program (einen Script) gestartet. Folgende Schritte werden ausgeführt:
  1. Die Website wird auf IPFS hochgeladen. Sobald der Upload fertig ist, speichert das Program den neuen CID.
  2. Die speziefische BCH-Wallet, welche mit dem Memo Account auf memo.cash verknüpft ist, wird instanziert.
  3. Eine Transaktion wird gebaut, in welcher der neue CID der Website als Meta-Information via Memo geschrieben wird.
Ist dieser Prozess erfolgreich abgeschlossen, wird auf dem öffentlichen Memo Profil nun der neue IPFS CID ersichtlich. Sobald nun erneut die Webseite zensurresistent.ch aufgerufen wird, wird man auf die neuste Version der IPFS Website weitergeleitet.

Was ist mit Updates?

Da der CID aus dem Inhalt der Daten resultiert, wird die URL der IPFS Website natürlich bei jedem Update der Seite komplett neu. Wie kann trotzdem sicher gestellt werden, dass zu jedem Zeitpunkt die aktuelle Version geladen wird?
Um dieses Problem zu lösen läuft beim laden der IPFS Website im Hintergrund ein Script, der nach ähnlichem Mechanismus wie oben beschrieben, auf der BCH Blockchain den neusten CID hohlt. Falls sich der User auf einer alten Version der Webseite befindet, wird ein Banner angezeigt, welcher auf die aktuellste Version verlinkt.
